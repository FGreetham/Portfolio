Allotment Adventure

Simple, "open-world" game where the player can do whatever they wish. There are no set objectives, but all player interactions in the garden are tracked and saved. Built using Unity and C#.

Github

Gameplay Video

Object Oriented Programming

Using principles of OOP, I tried to incorporate a seperation of concerns within my scripts and implemented a class for player interactions, which could be inherited across my classes that dealt with any kind of object interaction through raycasting. This is useful for scaling up in the future if I want to add more objects.

I also refactored my code to make it less tightly coupled, so I could push the variables straight to my 'DataManager' script when influenced by the player. This reduced the need for other scripts to reference each other and I could collate the player interactions in one place from the start.

Data Collection and JSON

The variables updated based on player interaction where hosted in a singleton class 'DataManager', as referenced above. This was then serialized into a JSON file which can be accessed by the player, or ideally, whoever requires this infomation on the player.

I wanted to learn how to push player interaction variables to an external file as I think on a grander scale it's useful to pick out data for understanding play styles, areas that players focus on or don't interact with at all and much more.

Questmaster

A large part of my focus was on my NPC and the side quests the player can choose to partake in. There are 3 simple tasks presented and I tracked if the player started the task or clicked 'No' to it. When the player fulfills a task they have started, that index is tracked and that completed task will no longer show up in the list of optional quests from the NPC.

The tasks will continuously loop around if the player says no.

The tasks will be removed from the list if they complete it. Once all tasks are complete, they get a different message box and this is tracked.

Close the Door

The only task given to the player from the beginning was to close and lock the shed door. I was inspired by The Door Problem in game design and wanted to explore a small part of it with regard to doors being closed and locked by the player.
